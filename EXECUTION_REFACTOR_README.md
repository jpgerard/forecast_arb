# Execution Refactor: Review → Intent → Execute

## Overview

This refactor cleanly separates **analysis/review** from **execution**, enabling a safe human-in-the-loop workflow:

1. **Run daily analysis** (`run_daily.py`) → generates review pack + candidates
2. **Review results** externally (manual + ChatGPT)
3. **Select a trade** and package it as an `OrderIntent`
4. **Execute the intent** via separate script (`execute_trade.py`)

At no point does the daily analysis script connect to IBKR for execution.

---

## What Changed

### PART A: `scripts/run_daily.py` Modifications

#### 1. **Deprecation Warning** (SAFETY)
- Added deprecation warning when `--submit` is used in `mode=prod`
- Warns user to use new `execute_trade.py` flow instead
- Preserves existing submission capability (with warning) for backward compatibility

#### 2. **New OrderIntent Emission** (NEW CAPABILITY)
- Added `--emit-intent` flag to package a trade for later execution
- Selection options:
  - `--pick-rank INT` - select by candidate rank
  - `--pick-expiry YYYYMMDD --pick-long STRIKE --pick-short STRIKE` - select by exact match
- Customization options:
  - `--limit-start FLOAT` - starting limit price (default: 92% of model debit)
  - `--limit-max FLOAT` - maximum limit price (default: 95% of model debit)
  - `--qty INT` - quantity (default: 1)
  - `--guard-max-spread-width FLOAT` - guard for spread width (default: 0.10)
  - `--guard-require-executable-legs` - require executable pricing (default: True)
  - `--intent-out PATH` - custom output path (default: `<run_dir>/order_intent.json`)

#### 3. **OrderIntent JSON Schema**
```json
{
  "strategy": "crash_venture_v1",
  "run_id": "<run_id>",
  "symbol": "SPY",
  "expiry": "YYYYMMDD",
  "type": "vertical_put_debit",
  "legs": [
    {"action": "BUY", "right": "P", "strike": 590},
    {"action": "SELL", "right": "P", "strike": 570}
  ],
  "qty": 1,
  "limit": {
    "start": 0.35,
    "max": 0.36
  },
  "tif": "DAY",
  "transmit": false,
  "guards": {
    "min_dte": 30,
    "max_debit": 0.36,
    "max_spread_width": 0.10,
    "require_executable_legs": true
  },
  "created_utc": "ISO8601 timestamp"
}
```

**Key Rules:**
- `transmit` is **always false** in intents generated by `run_daily.py`
- Includes full audit trail via `run_id`
- Limit prices default to conservative percentage of model debit

### PART B: `forecast_arb/execution/execute_trade.py` (NEW FILE)

#### Responsibilities
1. Load and validate `order_intent.json`
2. Connect to IBKR (via `ib_insync`)
3. Qualify option contracts
4. Pull live bid/ask quotes for each leg
5. Build BAG combo contract
6. Enforce guards:
   - Max debit
   - Max spread width
   - Executable legs required
   - Min DTE
7. Place order as single combo
8. Write execution receipt JSON

#### CLI Flags
- `--intent PATH` (required) - path to `order_intent.json`
- `--live` or `--paper` (required, mutually exclusive) - execution mode
- `--transmit` (optional, default: false) - transmit order to exchange
- `--confirm SEND` (required if `--transmit`) - safety confirmation
- `--host HOST` (optional, default: 127.0.0.1) - IBKR host
- `--port PORT` (optional, default: 7496 live / 7497 paper) - IBKR port

#### Safety Features
- **NO silent fallbacks** - hard abort on any guard violation
- **Explicit confirmations required**:
  - Must specify `--live` or `--paper`
  - Must specify `--transmit` to send order to exchange
  - Must specify `--confirm SEND` when using `--transmit`
- **Clear logging** for every step

## New Workflow

### Example: Full Human-in-the-Loop Flow

#### Step 1: Run Daily Analysis with Review-Only Mode
```powershell
python scripts/run_daily.py `
  --review-only-structuring `
  --snapshot snapshots/SPY_snapshot_20260203_104450.json `
  --p-event-source kalshi-auto
```

**Output:**
- `review_pack.md` - human-readable analysis
- `review_candidates.json` - machine-readable candidates
- `gate_decision.json` - edge gate results
- `p_event_implied.json` - options-implied probability

#### Step 2: Review Results
- Open `review_pack.md`
- Analyze candidates, pricing, edge, confidence
- Decide if/which trade to execute

#### Step 3: Emit OrderIntent for Selected Trade
```powershell
python scripts/run_daily.py `
  --emit-intent `
  --pick-rank 1 `
  --qty 1 `
  --limit-start 12.50 `
  --limit-max 12.75 `
  --snapshot snapshots/SPY_snapshot_20260203_104450.json `
  --p-event-source kalshi-auto
```

**Output:**
- `order_intent.json` - machine-readable trade intent

#### Step 4: Execute Intent (Paper Trading)
```powershell
python -m forecast_arb.execution.execute_trade `
  --intent runs/crash_venture_v1/[run_id]/artifacts/order_intent.json `
  --paper
```

**Behavior:**
- Connects to IBKR paper account (port 7497)
- Qualifies contracts
- Fetches live quotes
- Enforces guards
- **Stages order** (does NOT transmit)

#### Step 5: Execute Intent (Live Trading with Transmit)
```powershell
python -m forecast_arb.execution.execute_trade `
  --intent runs/crash_venture_v1/[run_id]/artifacts/order_intent.json `
  --live `
  --transmit `
  --confirm SEND
```

**Behavior:**
- Connects to IBKR live account (port 7496)
- Qualifies contracts
- Fetches live quotes
- Enforces guards
- **Transmits order** to exchange

### run_daily.py
- ✅ **NEVER connects to IBKR for execution**
- ✅ **NEVER places, stages, or transmits orders**
- ✅ Shows deprecation warning if `--submit` used in `mode=prod`
- ✅ All existing functionality preserved (review packs, gating, structuring, etc.)

### execute_trade.py
- ✅ **Requires explicit `--live` or `--paper`**
- ✅ **Requires explicit `--transmit` to send orders**
- ✅ **Requires `--confirm SEND` when transmitting**
- ✅ **Hard aborts on guard violations** (no silent fallbacks)
- ✅ All IBKR execution logic isolated here

## Guards Enforced in execute_trade.py

1. **max_debit**: Abort if live debit exceeds specified maximum
2. **max_spread_width**: Abort if spread width (as % of spot) exceeds limit
3. **require_executable_legs**: Abort if any leg missing bid/ask quotes
4. **min_dte**: Abort if days to expiry below minimum

All guards cause **hard failure** with clear error messages.

## Benefits of This Refactor

### Safety
- **Structural separation** prevents accidental execution during analysis
- **Explicit confirmations** at every dangerous step
- **Guard enforcement** at execution time with live data
- **Clear audit trail** via OrderIntent JSON

### Clarity
- **Single responsibility** per script
- **Predictable behavior** - no hidden execution paths
- **Easy to understand** workflow

### Flexibility
- **Human review** before execution
- **External tools** can review OrderIntents
- **Automated execution** still possible (via scripting `execute_trade.py`)
